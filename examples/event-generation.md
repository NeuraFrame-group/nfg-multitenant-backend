# Example – Dynamic Event Page Generation

## Scenario
Un produit peut contenir plusieurs événements (masterclass, formation, live).

## Behaviour
- Si une date est fournie via l’URL, la page affiche l’événement correspondant
- Sinon, le système sélectionne automatiquement le prochain événement à venir
- La page reste générique et réutilisable

## Benefit
- Une seule page pour plusieurs événements
- Moins de configuration manuelle
- Réduction des erreurs humaines
